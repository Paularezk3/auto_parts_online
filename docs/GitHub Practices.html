Branch Naming Conventions

Maintain clear and professional branch names for better collaboration and organization.

  • Use lower case and hyphens (-) as separators.
  • Include Jira issue keys in the branch name for traceability.
  • Prefix branch names with categories like feature/, bugfix/, or hotfix/.

Examples:

  • feature/PROJ-123-add-login-screen
  • bugfix/PROJ-456-fix-cart-issue
  • hotfix/PROJ-789-critical-api-bug
  • release/v1.2.3

Commit Message Guidelines

Write professional and descriptive commit messages to improve traceability and understanding.

Format:

        <type>(<scope>): <short summary>
        <empty line>
        <longer description explaining what changed and why>
        <empty line>
        <references>

Commit Types:

  • feat: A new feature.
  • fix: A bug fix.
  • docs: Documentation updates.
  • style: Code formatting changes (no functional changes).
  • refactor: Code refactoring without functional changes.
  • test: Adding or fixing tests.
  • chore: Non-code updates (e.g., build scripts).

Examples:

  • feat(PROJ-123): add login functionality
  • fix(PROJ-456): resolve cart quantity bug
  • docs(PROJ-789): update README with deployment steps

Using Smart Commits

Smart commits allow you to directly interact with Jira issues from your Git commit messages. Include specific commands in your commit messages to update Jira tasks.

Smart Commit Syntax:

  • [Jira Issue Key] [Command] [Optional Comment]

Available Commands:

  • #close: Marks the issue as closed or resolved.
  • #done: Marks the issue as done.
  • #in-progress: Moves the issue to the "In Progress" status.
  • #review: Requests a review for the issue.

Examples:

  • PROJ-123 #close Fixed the login authentication bug.
  • PROJ-456 #in-progress Implemented basic product search functionality.
  • PROJ-789 #done Updated README and added deployment instructions.

Best Practices:

  • Ensure the issue key is accurate to link your commit to the correct task.
  • Keep the command relevant to the current progress of the issue.
  • Write meaningful comments to provide context for the commit.

Creating Professional Jira Tasks

Jira tasks should be detailed, actionable, and aligned with Scrum cycles to ensure smooth progress tracking and effective team collaboration.

Task Creation Steps:

  • Step 1: Define the Task Title
    • Use a short and descriptive title (e.g., "Implement Login Screen").
    • Include relevant context, such as feature name or bug scope.
  • Step 2: Write a Detailed Description
    • Explain the purpose and outcome of the task.
    • Include technical details, mockups, or links to resources if applicable.
  • Step 3: Add Acceptance Criteria
    • Define measurable outcomes to mark the task as complete.
    • Example: "The login screen should validate email/password and display error messages."
  • Step 4: Assign Priority and Estimate Effort
    • Use priority levels like "High," "Medium," or "Low."
    • Estimate story points based on the complexity and effort required.
  • Step 5: Link Dependencies
    • Connect related tasks or subtasks.
    • Example: Link "Create API for Login" to "Implement Login Screen."

Task Types:

  • Story: High-level functionality or user requirements (e.g., "As a user, I want to add items to my cart").
  • Task: A specific piece of work (e.g., "Implement cart functionality").
  • Bug: An issue to be fixed (e.g., "Fix checkout page crash on mobile").
  • Epic: A large body of work divided into multiple stories (e.g., "E-commerce Shopping Cart").

Task Alignment with Scrum Cycles:

  • Ensure all tasks are tied to the sprint goal.
  • Break down stories into manageable tasks that can be completed within a sprint.
  • Regularly update the task status to reflect progress (e.g., "To Do," "In Progress," "Done").

Examples of Well-Defined Tasks:

  • Title: Implement Login Screen
  • Description: Create a Flutter-based login screen that includes fields for email and password, a login button, and validation logic.
  • Acceptance Criteria:
    • Valid email addresses are accepted; invalid ones display error messages.
    • Login fails with incorrect credentials and displays an error message.
