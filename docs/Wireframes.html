Home Page (Mobile Application)

Purpose

The Home Page for the mobile application acts as a central hub for users to explore featured content, browse categories, and access primary functionalities with a clean and intuitive layout designed for smaller screens.

Components

  • Header:
    • Logo: Centered at the top of the screen for branding.
    • Notification Icon: Positioned at the top-right for quick access to notifications.
  • Bottom Navigation Bar:
    • Tabs: Home, Categories, Cart, Profile, and More.
    • Icons and Labels: Simple, intuitive icons with labels for clarity.
  • Hero Section:
    • Small carousel slider displaying promotions or featured products.
    • Swipe Gesture: Allows users to swipe through banners.
  • Featured Products Section:
    • Horizontal Scroll: Displays product cards in a scrollable row.
    • Product Cards: Include image, title, price, and a compact "Add to Cart" button.
  • Categories Section:
    • Category List: A grid of categories with icons or images and labels.
    • Optimized for touch: Large tappable areas for easy navigation.
  • Call-to-Action Section:
    • Sign-up/Newsletter Prompt: A banner encouraging user sign-ups with a single tap action.

Layout

Mobile:

  • Top Header: Compact with essential icons like notifications.
  • Bottom Navigation: Persistent and easily accessible with intuitive tabs.
  • Vertical Scrolling: Smooth, single-column layout optimized for one-handed use.
  • Touch-friendly Elements: Buttons and cards designed for quick interaction.
Desktop:
  • Header spans the full width, with navigation and search bar centrally aligned.
  • Hero section occupies the top fold with a visually striking banner.
  • Content sections are divided into grids, ensuring balance and readability.
  • Footer spans the full width with a clean, organized structure.

Products Listing Page

Purpose

Enable users to browse and filter car spare parts effectively. This page provides tools for filtering products based on various criteria such as categories, price range, brands, and compatibility with specific car makes and models. The design supports accessibility, usability, and multi-lingual usage in English and Arabic. It adapts to dark and light mode preferences.

Components

  • Filters Panel:
    • Category: A dropdown or checkbox list for selecting product categories such as Engine, Body Parts, Accessories, etc. Multi-select functionality is included to enhance user experience.
      • - When the user clicks the "Select Categories" button, a pop-up dialog opens with a list of checkboxes. - The user can select one or more categories. - After pressing "Done," the selected categories are dynamically applied as filters, updating the product list in real-time.
    • Price Range Slider: Allows users to specify minimum and maximum price values. Includes a text input for precise entry.
      • - As the user adjusts the slider, the displayed price range updates in real-time. - The product list is filtered dynamically to show only items within the selected price range.
    • Brand Selector: Dropdown menu featuring major car brands with a type-ahead search for quick selection.
      • - When the user types into the dropdown's search bar, matching brand names are suggested. - Selecting a brand dynamically filters the product list to include only items from the selected brand.
    • Compatibility Selector: A dropdown for selecting car make, model, and year. Ensures accuracy and usability with dependency filtering (e.g., model options depend on selected make).
      • - When a car make is selected, the models dropdown is populated with relevant options. - Once a model is chosen, the year dropdown adjusts to display only valid years for that model. - All selections dynamically filter the product list, showing only compatible parts.
  • Product Cards: Displayed in a responsive grid or list view. Each card contains:
    • Product image with zoom functionality on hover.
    • Title, short description, and price prominently displayed for quick decision-making.
    • Action buttons for "Add to Cart" and "View Details." Includes a wishlist icon.
    • Indicators for stock availability (e.g., "In Stock," "Out of Stock").
  • Pagination: Includes numbered buttons, "Next" and "Previous" options, and optional infinite scrolling with a loading spinner.

Layout

The layout is designed to accommodate both desktop and mobile users:

  • Desktop: Filters are displayed in a vertical sidebar on the left. Products are arranged in a multi-column grid to utilize available screen space.
  • Mobile: Filters are collapsible and accessible via a button at the top. Products are displayed in a single-column list to maximize readability on smaller screens.
  • Dark and Light Modes: The page dynamically adjusts its styling based on user preferences. Uses WCAG-compliant contrast ratios to ensure readability.

Accessibility Features

  • Full keyboard navigation support for all interactive components (e.g., dropdowns, sliders, buttons).
  • ARIA roles and labels for screen readers.
  • High-contrast mode compatibility.
  • Multi-lingual support with a language toggle for English and Arabic, with right-to-left (RTL) styling for Arabic text.

User Interaction Details

Filters and selections dynamically update the product list in real-time without requiring a full page reload, ensuring a seamless user experience.

Product Details Page

Purpose

Provide users with comprehensive information about a specific product, enabling them to make informed purchasing decisions. Includes details such as images, descriptions, compatibility, pricing, and customer reviews.

Components

  • Product Image Gallery: A carousel with zoom and swipe capabilities. Supports multiple images and videos where applicable. Includes alt text for accessibility.
  • Product Details Section:
    • Title: Prominent display with clear typography.
    • Price: Includes discount pricing and clear formatting (e.g., crossed-out original price).
    • Stock Availability: Real-time stock updates with visual indicators (e.g., "In Stock" in green, "Out of Stock" in red).
    • Compatibility Information: Highlight compatible car makes, models, and years in a distinct, easily noticeable section. Use bold text or colored tags (e.g., badges with car icons) to represent supported car models clearly.
    • Description: A comprehensive overview of the product, including technical specifications and benefits.
  • "Add to Cart" Button: A large, prominent button with hover and focus states. Includes an "Added to Cart" confirmation.
  • Customer Reviews Section: Displays a list of customer reviews with:
    • Average star rating at the top.
    • Individual review text and customer names.
    • Ability to filter reviews by rating (e.g., 5-star, 4-star).
  • Service Center Integration:

    Display a clear "Find a Service Center" call-to-action below the compatibility information.

    • Text: A descriptive label like "Locate a Trusted Service Center Near You."
    • Style: Place this text as a button-like element styled for easy clickability.
    • Action: Clicking this will open a page or map where users can input their location or allow location access to find nearby trusted service centers. The results should show distance, contact details, and service ratings, if available.

  • Warranty Information:

    Display the warranty terms as a badge-style element.

    • Appearance: The badge should include a small warranty icon (e.g., a shield or document) and text like "1-Year Warranty Included."
    • Actionable Info Icon: Beside the badge, place an info icon (e.g., a small "i" inside a circle). When clicked, this icon opens a bottom modal sheet containing detailed warranty information, including coverage, exclusions, and instructions for claims.
    • Modal Details: Ensure the modal is scrollable, has a close button, and includes contact information for warranty inquiries.

Layout

Desktop: A two-column layout with:

  • Image gallery prominently displayed on the left.
  • Product details, price, and actions (e.g., "Add to Cart") on the right.
Mobile: A single-column layout with a vertically stacked arrangement for ease of scrolling.

Cart Page

Purpose

The Cart Page allows users to review their selected products, update quantities, explore recommendations, and proceed to checkout.

Components

  • Product List Section

    The Product List Section includes individual product cards for each item in the cart:

    • Product Card (Cart Item):
      • Image: Positioned on the left side of the card, ensuring clear representation and responsive resizing.
      • Details Column: Positioned to the right of the image and includes:
        • Product Name: Displayed in bold text. Clicking the name navigates to the Product Details Page.
        • Price: Shows the original price (strikethrough) and discounted price (highlighted).
        • Quantity Selector: Features increment (+) and decrement (-) buttons inline beside the quantity value. When the quantity is set to 1, the decrement button transforms into a trash icon, removing the product from the cart.
        • Touch Action: The entire card is tappable, linking to the Product Details Page.

  • Cart Summary Section

    The Cart Summary Section provides a detailed breakdown:

    • Subtotal: Total of product prices before tax.
    • Tax: Displayed separately.
    • Total: Final price (Subtotal + Tax).
    • "Proceed to Checkout" Button: Prominently displayed, fixed at the bottom of the screen on mobile devices for accessibility.

  • Products For You Section

    Displays a recommendation carousel or grid:

    • Horizontally scrollable list of recommended products.
    • Each item includes an image, product name, price (original and discounted), and an "Add to Cart" button.
    • Accessible from both desktop and mobile views.
    • Heading: Clearly labeled as “Products for You.”

Layout

  • Desktop View

    Two-column layout:

    • Left Column: Product List.
    • Right Column: Cart Summary and Checkout Button, aligned vertically.

  • Mobile View

    Stacked layout:

    • Product List appears first, followed by the Cart Summary.
    • Cart Summary and "Proceed to Checkout" button are pinned to the bottom for better accessibility.

    Additional features for mobile:

    • Condensed Product Cards:
      • Compact design with reduced padding and smaller font sizes.
      • Align image and text in a single row for minimal scrolling.
    • Sticky Cart Summary:
      • Collapses into a floating button (e.g., "View Total") at the bottom when scrolling through the Product List.
      • Expands into full details when tapped.
    • One-Tap Usability: Optimized touch targets for buttons to ensure ease of use with one hand.

Checkout Page

Purpose

The checkout page is designed to guide the user through the final steps of their purchase journey. This page ensures seamless collection of necessary information, including shipping and payment details, while providing a clear review of the order to build user confidence before completing the transaction.

Components

  • Shipping Address Form: A user-friendly form for entering the recipient's name, address, and contact details. The form should validate inputs in real-time (e.g., zip code format, phone number length) and support autofill to enhance usability.
  • Payment Method Selector: A selection interface offering multiple payment options (e.g., Credit/Debit Card, PayPal, Digital Wallets). Includes fields for securely capturing card details or redirecting to external payment gateways as needed.
  • Order Review Section: A summary box displaying the list of items in the cart with itemized details (name, quantity, price), applicable discounts, shipping fees, and the total amount. Includes an "Edit Cart" link for quick updates.
  • Points Redemption:
    • "Use Points" Toggle: An interactive toggle or checkbox allowing users to apply reward points for discounts. Clearly indicates the total points available and how many are being used.
    • Real-time Update: Displays the updated total cost dynamically when points are applied. Example: "Total: $45 after applying 500 points".
  • Place Order Button: A prominently styled button to finalize the purchase. Should display a spinner or loading indicator after clicking to indicate the process is ongoing.
  • Promotional Code Field: An optional input field where users can enter promo codes. Includes a "Apply Code" button to validate and apply discounts.
  • Security Assurance Badges: Visual indicators (e.g., "Secure Checkout," SSL badges) to reassure users about the safety of their payment details.

Layout

Desktop: A clean and intuitive three-column layout with the following arrangement:

  • Left Column: Shipping address form
  • Middle Column: Payment method selector
  • Right Column: Order review and summary

Mobile: A multi-step form optimized for smaller screens. Each section (shipping, payment, order review) is displayed on a separate screen with "Next" and "Back" navigation buttons for progression.

Accessibility

Ensure compliance with WCAG 2.1 standards:

  • All form fields have clear labels and accessible descriptions.
  • Keyboard navigation is supported across all components.
  • Proper focus management ensures users are directed to the correct field after an error.
  • Color contrast ratios meet accessibility standards for readability.

Analytics Page (Admin)

Purpose

Provide suppliers with actionable insights on sales performance, product trends, and inventory turnover to support data-driven decisions.

Components

  • Sales Graphs:
    • Line/Bar charts showing sales over time (daily, weekly, monthly).
    • Filters: Date range, product categories, or specific products.
  • Top-Performing Products:
    • List of products ranked by sales volume.
    • Include metrics like revenue, number of units sold, and profitability.
  • Inventory Turnover:
    • Chart or table showing inventory movement and restocking frequency.
  • Customizable Reports:
    • Option to generate and export PDF/Excel reports.
  • Loyalty Program Management:
    • Points Rules Management:
      • Set earning rates (e.g., points per $1).
      • Configure referral bonuses.
    • Tiers Management:
      • Define thresholds and benefits for Silver, Gold, and Platinum tiers.
      • Set automatic progression rules.
    • Reward Catalogue:
      • Add, update, or remove redeemable rewards.
      • Set point costs for rewards.
    • User Analytics:
      • Track loyalty program engagement and tier distribution.
      • Monitor rewards redemption patterns.

Layout

Desktop: Interactive charts and tables in a dashboard-style layout.
Mobile: Simplified, scrollable view with expandable graphs and summaries.
